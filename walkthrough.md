# Code2Course：架構與運作原理解析

這份文件清楚說明了 **Code2Course** 是一個什麼樣的工具，以及它在底層究竟是「如何」將冰冷的程式碼轉換為生動的教學影片的。

---

## 1. Code2Course 做了什麼？

**Code2Course** 是一個全自動化的「原始碼轉教材」生成引擎。
它的核心目標是：當開發者完成一個軟體或韌體專案後，完全**不需要人工寫簡報、錄音或剪輯**，只要啟動這個工具，它就能自動分析專案理的程式碼脈絡，並產出：

1. **結構化腳本 (JSON)**：將專案拆解成有邏輯連貫性的教學段落。
2. **課程簡報 (PPTX)**：自動排版出供老師授課用的投影片。
3. **影音教學影片 (MP4)**：將簡報視覺化，並配上 AI 合成語音解說，形成最終的微課程影片。

---

## 2. 它是怎麼做到的？ (核心三階段架構)

整個程式 (`main.py` 內的 `Code2Course` 類別) 將這個艱鉅的任務解耦為三個可獨立運作的階段：

### 📍 [階段 1] 專案掃描與 Prompt 封裝
- **運作原理**：
  程式會根據 `code2course_config.json` 中的 `supported_exts` (支援副檔名) 與 `exclude_dirs` (黑名單，如 `node_modules`) 去掃描目標資料夾。
- **防止超載機制**：在爬取程式碼時，設有 30 萬字元的安全上限，防止送交給 AI 時超出 Token 限制。
- **產出**：它會把所有掃描到的程式碼，與外層的教學指令樣板 (`code2course_prompt.txt`) 組合，最後在佔存區產生一份 `gemini_prompt_record.txt` 檔案。

### 📍 [階段 2] 呼叫大語言模型與簡報生成
- **運作原理**：
  讀取第一階段產生好的完整 Prompt 字串，透過 Google 的 `google.generativeai` 套件，呼叫 **Gemini 2.5 Flash** 模型。
- **Prompt Engineering 技巧**：我們在內部強制要求 AI 必須以純 JSON Array 格式回傳，且必須包含「全局開場」、「段落轉場」與「完美總結」，讓產出具有真實講師的連貫性。
- **產出**：
  1. 將 AI 回傳的 JSON 腳本存為 `gemini_analysis_result.json`。
  2. 呼叫 `python-pptx` 套件，將 JSON 裡面的 `title` 和 `bullet_points` 動態寫入空白投影片中，生成 `[專案]_course_slides.pptx`。

### 📍 [階段 3] 語音生成與影音合成
- **運作原理**：這是一個不需要外部簡報軟體的純粹多媒體合成階段，完全依賴 Python 視覺與音訊庫。
  1. **語音生成 (`edge-tts`)**：讀取 JSON 腳本中的 `script` 欄位，透過微軟的 Edge TTS 指令列工具，生成流暢擬真的中文語音檔 (`.mp3`)。
  2. **畫面繪製 (`Pillow`)**：不依賴剛才產生的 PPT，而是直接使用 Python 的 `PIL` (Pillow) 函式庫，使用黑色深色背景結合藍色標題橫幅，並動態抓取作業系統的字型 (如 Mac 的 PingFang 或 Linux 的 NotoSansCJK)，繪製出精美的分鏡圖片 (`.png`)。
  3. **影音融合 (`MoviePy`)**：使用 `moviepy` 函式庫，將產出的 圖片 (`.png`) 根據對應錄音檔 (`.mp3`) 的長度拉長為影片片段 (`VideoClip`)，最後將所有片段串接 (concatenate)，轉檔輸出。
- **產出**：最終流暢的 `[專案]_course_video.mp4` 影片。

## 3. 自動化與靈活性的秘密武器

為了讓這套系統能夠應付各種情境（本地測試 vs. 雲端全自動化），我們實作了以下防呆與靈活機制：

- **時間戳與斷點接關機制 (Multi-Mode CLI)**：
  透過 `questionary` 套件打造豐富的終端機互動選單。只要執行過一次掃描，進度都會依照「時間戳記」被保存在 `outputs/` 目錄下。開發者可以隨時單獨啟動 [階段 2] 或 [階段 3]，程式會自動去歷史資料夾撈取上一關的進度，大幅省去重複掃描或付費給 API 的時間。

- **Headless 無介面設計 (GitHub Actions 支援)**：
  當這套工具被掛載在 GitHub 的 CI/CD 環境時，沒有人可以按鍵盤選擇專案。因此我們加入了特別的環境變數監聽器 `os.getenv("AUTO_MODE")`。只要偵測到這個變數，程式就會強烈覆蓋掉所有的互動選單，切換為「靜默一條龍模式」，這正是它能在任何伺服器背景自動吐出影片的關鍵。
