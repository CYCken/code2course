# Code2Course：架構與運作原理解析

這份文件清楚說明了 **Code2Course** 是一個什麼樣的工具，以及它在底層究竟是「如何」將冰冷的程式碼轉換為生動的教學影片的。

---

## 1. Code2Course 做了什麼？

**Code2Course** 是一個全自動化的「原始碼轉教材」生成引擎。
它的核心目標是：當開發者完成一個軟體或韌體專案後，完全**不需要人工寫簡報、錄音或剪輯**，只要啟動這個工具，它就能自動分析專案裡的程式碼脈絡，並產出：

1. **結構化腳本 (JSON)**：將專案拆解成有邏輯連貫性的教學段落。
2. **多樣化素材 (Images/Videos)**：根據教學內容自動抓取高畫質視覺元素。
3. **課程簡報 (PPTX)**：自動排版出專業、優雅且符合教學邏輯的投影片。
4. **影音教學影片 (MP4)**：將簡報視覺化，並配上 AI 合成語音解說，形成最終的微課程影片。

---

## 2. 五階段模組化架構 (The 5-Stage Pipeline)

Code2Course 將繁重的影音產出任務解耦為 5 個可獨立運作、隨時接關的階段：

### 📍 [階段 1] 專案掃描與 Prompt 封裝
- **運作原理**：
  程式會根據 `code2course_config.json` 中的支援副檔名與排除資料夾 (如 `node_modules`, `drivers`) 去掃描目標專案。
- **防止超載機制**：在爬取程式碼時，設有 **30 萬字元的預設安全上限** (可從 `code2course_config.json` 中調整)，防止送交給 AI 時超出 Token 限制。
- **產出**：將程式碼與教學指令樣板 (`code2course_prompt.txt`) 組合，產生 `gemini_prompt_record.txt`。

### 📍 [階段 2] 兩階段強固型分析 (Two-Pass Analysis)
- **核心與媒體解耦**：為了解決大規模專案的 JSON 截斷與損毀問題，我們捨棄了單次輸出的臃腫模式，改採兩階段架構：
  1. **Pass 1: 核心內容分析 (Core Brain)**：由 AI 讀取專案全貌，產出精簡且穩定的 JSON。包含：講稿、標題、重點、Marp 簡報結構。
  2. **Pass 2: 媒體屬性增強 (Media Enrichment)**：根據 Pass 1 的結果，AI 以 **5 頁為一組** 進行分批細節寫作（產生 InVideo 分鏡描述與 Remotion 動畫參數）。
- **優勢**：
  - **極致穩定性**：大幅降低單次請求的 Token 負荷，徹底根治 `Unterminated string` (JSON 損毀) 錯誤。
  - **輸出精度提高**：AI 在增強階段專注於視覺與動畫描述，內容品質更專業。
- **產出**：核心腳本 (`gemini_analysis_result.json`) 以及獨立的工具專用檔 (`invideo.json`, `remotion_data.json`)。

### 📍 [階段 2-1] 媒體增強恢復與迭代 (Enrichment Sub-Stage)
- **智慧節流機制**：當核心分析已成功，但您想更換 InVideo 風格，或初次生成時出錯，您可以啟動階段 2-1。
- **運作原理**：直接從暫存區撈取 `gemini_analysis_result.json`，跳過耗時的原始碼掃描與核心分析，直接針對內容進行二次媒體增強。
- **優勢**：在處理如 35 頁以上的大型專案時，能省下大量的 AI 等待時間 (省去 90% 以上的重跑耗時)。

### 📍 [階段 3] Pexels 素材抓取與管理
- **運作原理**：
  讀取 JSON 腳本中的視覺建議 (Visual suggestions)，透過 Pexels API 自動搜尋並下載對應的高畫質圖片與影片。
- **精細化管理機制**：
  - **更新模式**：提供「跳過已存在」、「全部重新下載」或「手動勾選特定分頁」三種模式，方便針對單一不滿意頁面進行局部更換。
  - **自動隨機化**：在重刷素材時，系統會自動在搜尋結果中進行隨機跳頁 (Page 1-10)，確保視覺素材不重複。
- **版權標註**：程式會同步記錄圖片攝影師資訊，供後續自動生成引用頁面。

### 📍 [階段 4] 專業簡報渲染 (PPTX)
- **運作原理**：
  利用 `python-pptx` 套件讀取 `code2course_style.json` 中的各項設計參數（如字級、RGB 顏色、遮罩透明度等）。
- **進階排版**：
  - **全螢幕底圖**：採取現代、大氣的全螢幕排版模式。
  - **內容遮罩**：自動在文字塊後方加入 50% 透明度的深色遮罩，確保閱讀性。
  - **智慧溢出防止**：標題若過長會自動換行並縮小字級，防止元素重疊。
- **產出**：最終的 `[專案]_course_slides.pptx` 投影片。

### 📍 [階段 5] 影音合成 (MP4)
- **運作原理**：這是一個不需要外部簡報軟體的純粹多媒體合成階段，完全依賴 Python 視覺與音訊庫。
  1. **語音生成 (`edge-tts`)**：將 JSON 腳本轉換為擬真的中文語音檔 (`.mp3`)。
  2. **畫面繪製 (`Pillow`)**：直接使用 Python 的 `PIL` (Pillow) 函式庫，採用 **圖層融合 (Alpha Composite)** 技術，繪製出帶有半透明文字遮罩的精美影格圖片。
  3. **合成影音 (`MoviePy`)**：具備 **版本自動相容機制** (同時支援 MoviePy 1.x 與 2.x API)，對齊語音與影格長度並渲染。
- **視覺優化**：
  - **Card Effect 遮罩**：在影片中同樣實作了 **50% 透明度 (Alpha 127)** 的背景遮罩，即便在複雜大圖下文字依然清晰。
  - **統一化邏輯**：當分頁無背景圖時，系統會自動「隱藏遮罩框」並「統一底色」，確保極致的視覺簡潔感。
- **產出**：播放流暢的 `[專案]_course_video.mp4` 教學影片。

## 3. 防呆與靈活機制

- **時間戳與斷點接關機制 (Checkpoint System)**：
  透過 `questionary` 套件打造豐富的選單。所有的產出都以時間戳記保存在 `outputs/` 下。使用者可以隨時單獨啟動各階段，程式會自動撈取進度。
- **智慧接力模式 (Stage Chaining)**：
  當完成任一獨立階段，系統會主動詢問是否接續執行下一關。若選擇「是」，則會自動攜帶專案狀態無縫推進，大幅節省操作時間。
- **全方位樣式引擎 (Universal Style Engine)**：
  不須修改核心程式碼，透過 `code2course_style.json` 即可同時控制 PPT 的排版細節以及影片的所有視覺參數 (Banner 顏色、變暗透明度、遮罩色值、字體大小等)。
- **Headless 無介面設計 (CI/CD 支援)**：
  當偵測到環境變數 `AUTO_MODE` 時，程式會切換為「靜默一條龍模式」，以便在 GitHub Actions 等背景環境自動產出影片。
- **自動標註 (Attribution)**：
  自動在簡報最後一頁生成 **References & Attributions**，尊重攝影師版權。
